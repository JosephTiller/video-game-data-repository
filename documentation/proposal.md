# Proposal: Video Game Data Repository

1. Rationale for the development of the system
  - User group: Gamers, game researchers, and data analysts who want structured information about Minecraft and Fortnite.
  - Information needs: Easy access to game data like sales, mobs, weapons, and seasons.
  - Gaps in information: Existing sources are scattered, unstructured, or incomplete; this repo organizes it all in one place.
  - Evaluation criteria / use cases:
    - Lookup total copies sold for Minecraft games.
    - Check hostile mobs in Minecraft and their health.
    - Find Fortnite weapons by type and rarity.
2. Specifications for the system
  - Folder structure:
    - game-data – contains Minecraft and Fortnite datasets
    - documentation – contains metadata guide, README, and proposal
  - Files: CSV datasets with metadata guide.
3. Statement of objectives
  - Store and organize Minecraft and Fortnite datasets.
  - Provide clear metadata for each dataset.
  - Make the repository easy to navigate and use.
4. Description of how the system will be built or developed
  - Built on GitHub with CSV files and Markdown documentation.
  - Metadata guide explains all fields.
  - Folder structure makes datasets easy to find.
5. Plans for field testing and dissemination
  - Public GitHub repository allows users to access and explore the data.
  - Feedback can be collected via GitHub issues for improvements.
6. Approximate timetable and skills needed
  - Skills: Basic GitHub, CSV formatting, Markdown writing.
  - Timeline: Can be implemented in 1–5 days.
